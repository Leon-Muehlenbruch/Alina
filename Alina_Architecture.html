🌐 Nostr-Protokoll — Dezentrales Relay-Netzwerk
Event-Typen (Publish & Subscribe)
Kind 4 Direktnachrichten (E2E verschlüsselt) NIP-04
Kind 42 Gruppennachrichten (Klartext) NIP-28
Kind 10420 Einladungscodes (SHA-256 Hash) Custom
Subscriptions auf jedem Relay
REQ dm-sub  { kinds: [4], "#p": [myPubkey] }
REQ room-sub { kinds: [42], "#e": [roomHash...] }
DMs gefiltert nach eigenem pubkey. Rooms gefiltert nach Room-Hash.

🛡 Sicherheit & Features
E2E-Verschlüsselung
NIP-04: ECDH-Schlüsselaustausch + AES-256-CBC. Privater Schlüssel verlässt nie das Gerät.
secp256k1AES-256
Einladungscodes
6-stelliger Code → SHA-256-Hash auf Relay. 10 Min Gültigkeit. Relay sieht nie den Klartext-Code.
SHA-25610 min TTL
Selbstzerstörende Nachrichten
TTL-Feld im Message-Envelope. Countdown-Timer in der UI. Cleanup alle 1s per Interval.
30s / 5m / 1h
Bildkompression
Client-seitige Komprimierung. Max 500 KB als Base64 Data-URL direkt im verschlüsselten Payload.
max 500 KB
Offline-Warteschlange
Nachrichten bei Offline in localStorage gepuffert. Automatisches Flush bei Reconnect.
localStorage
PWA
Vite-Plugin-PWA (Workbox). CacheFirst für App-Shell. Installierbar auf iOS, Android, Desktop.
Service Worker
