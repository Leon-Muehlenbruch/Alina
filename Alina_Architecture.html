🌐 Nostr Protocol — Decentralised Relay Network
Event Types (Publish & Subscribe)
Kind 4 Direct messages (E2E encrypted) NIP-04
Kind 42 Group messages (plaintext) NIP-28
Kind 10420 Invite codes (SHA-256 hash) Custom
Subscriptions on each relay
REQ dm-sub  { kinds: [4], "#p": [myPubkey] }
REQ room-sub { kinds: [42], "#e": [roomHash...] }
DMs filtered by own pubkey. Rooms filtered by room hash.

🛡 Security & Features
E2E Encryption
NIP-04: ECDH key exchange + AES-256-CBC. Private key never leaves the device.
secp256k1AES-256
Invite Codes
6-digit code → SHA-256 hash on relay. 10 min validity. Relay never sees the plaintext code.
SHA-25610 min TTL
Self-Destructing Messages
TTL field in message envelope. Countdown timer in the UI. Cleanup every 1s via interval.
30s / 5m / 1h
Image Compression
Client-side compression. Max 500 KB as Base64 data URL directly in the encrypted payload.
max 500 KB
Offline Queue
Messages buffered in localStorage when offline. Automatic flush on reconnect.
localStorage
